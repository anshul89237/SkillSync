SkillSync Backend Architecture

Accurate service topology, boundaries, and patterns used in the current implementation.

1. Service Topology

Gateway and Infra
api-gateway, config-server, eureka-server
Core Domains
auth-service, user-service, skill-service, session-service
Async and Payments
notification-service, payment-service, rabbitmq, redis

2. Security Boundary

  • Gateway validates JWT and role.
  • Gateway propagates trusted headers: X-User-Id and X-User-Role.
  • Downstream services enforce authorization by role and ownership.

3. Data and Messaging

  • PostgreSQL for persistent transactional data.
  • Redis for cache-aside read acceleration.
  • RabbitMQ for event-driven communication and decoupling.
  • Zipkin, Prometheus, Grafana, Loki for visibility and diagnostics.

4. Pattern Summary

PatternWhere usedWhy
CQRSUser and other business servicesClear separation of reads/writes and simpler cache management
Cache-asideRedis query pathsFaster reads with graceful fallback to DB
Event-driven integrationPayment, notifications, cross-domain actionsLoose coupling and better fault isolation
Central config + discoveryConfig Server + EurekaConsistent service registration and runtime config

5. Current API Domains

  • Auth APIs: registration, login, OTP verify, reset password.
  • User APIs: profile, mentor onboarding, group operations.
  • Session APIs: request, accept/reject, complete, review.
  • Payment APIs: order creation, verification, business action orchestration.
